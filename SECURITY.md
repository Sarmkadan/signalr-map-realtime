# Security Policy

We take the security of this project seriously. Please follow the policy below to report any security vulnerabilities.

## Supported Versions

Security updates are currently provided for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly. We appreciate your efforts to keep our community safe.

Please use one of the following methods to report a vulnerability:

1. **GitHub Private Advisory (Preferred):** Use GitHub's built-in private vulnerability reporting feature to submit your findings securely.
2. **Email:** Alternatively, you can contact us directly at **rutova2@gmail.com**.

Please include a detailed description of the vulnerability, steps to reproduce it, and any potential mitigations if known.

## Response Timeline

We will investigate all legitimate reports and provide an **acknowledgment within 48 hours** of receiving your report. We will keep you updated on the progress of our investigation and any planned fixes.
